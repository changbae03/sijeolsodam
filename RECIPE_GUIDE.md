# 시절소담 레시피 작성 가이드

이 문서는 시절소담의 레시피를 작성할 때 지켜야 할 기준을 정리한 것입니다.
새로운 식재료를 확장하거나 기존 레시피를 다듬을 때 이 문서를 먼저 참고하세요.

---

## 1. 핵심 철학

> "시절소담은 레시피 웹사이트가 아니라, 세상에서 가장 좋은 제철 요리 가이드다."

- **양보다 질.** 레시피 개수를 늘리는 것 자체가 목표가 아니다.
- 모든 레시피는 **요리를 가르쳐야** 한다 — 무엇을 하는지(WHAT)뿐 아니라 왜 그렇게 하는지(WHY)까지.
- 글은 **경험 많은 셰프가 옆에서 가르쳐주는 듯한** 톤이어야 한다. 로봇처럼 딱딱하거나, AI가 쓴 것처럼 반복적이면 안 된다.
- 처음 요리하는 사람이라고 가정하고, 실수를 미리 막아준다.

---

## 2. 4단계 컬렉션 체계

식재료마다 난이도/목적이 다른 4개의 컬렉션으로 레시피를 분류한다.

| level | 라벨 | 설명 |
|---|---|---|
| `home` | 🌱 데일리 홈쿡 | 빠르고 쉬운 일상 요리. 처음 요리하는 사람이 시작하는 자리 |
| `weekend` | 🔥 주말 요리 | 시간 여유가 있을 때 도전하는 한 단계 더 깊은 요리 |
| `world` | 🌍 세계 요리 | 그 식재료가 들어가는 세계 각국의 정통 요리. `cuisineContext` 필수 |
| `chef` | 👨‍🍳 셰프 컬렉션 | 레스토랑급 기법·플레이팅을 보여주는 요리. `platingGuide` 필수 |

### 목표 개수
**식재료 하나당 4단계 각각 최소 5개**가 목표다. (현재 KAMIS 9개 핵심 식재료: 감자·사과·무·방울토마토·알배추·배·고구마·딸기·오이는 전부 달성. 가지·시금치도 5개씩 신규 구축 완료.)

신규 식재료를 확장할 때는:
1. 기존 home 레시피 개수를 먼저 확인한다 (3개 이상이면 그대로 두고 부족한 만큼만 추가).
2. weekend/world/chef는 보통 0~1개부터 시작하므로 각각 4~5개씩 새로 써야 한다.
3. 한 식재료를 5/5/5/5(총 20개)로 완성하는 게 한 번의 작업 단위로 적당하다.

## 3. 레시피 데이터 구조

### 3.1 기본 필드 (`Recipe`)
```
id, month, title, subtitle, category, difficulty, level,
cuisineContext?(world용), platingGuide?(chef용), masterclass?,
cookTime, servings, heroImage, mainIngredient, description,
ingredients[], steps[], tips[], youtubeQuery
```

- `mainIngredient`는 **정확한 식재료 표기**를 써야 한다 (예: 사과 레시피엔 `'사과'`, 햇감자 전용 레시피엔 `'햇감자'`). 식재료-레시피 매칭은 접미사(suffix) 기준이라, `'알배추'`는 `'배추'`로 끝나서 매칭되지만 `'배'`로는 끝나지 않아 매칭되지 않는다. **절대 `includes()` 식으로 막연히 글자가 포함되는지 가정하지 말 것** — 과거 "배" 레시피가 "알배추" 페이지에 잘못 노출된 버그의 원인이었다.

### 3.2 마스터클래스 콘텐츠 (`MasterclassContent`)
모든 레벨에 채우는 것을 기본으로 한다 (홈쿡도 예외 없음).

```
chefIntro          — 이 식재료가 왜 특별한지, 이 레시피가 왜 지금 좋은지
ingredientSelection — 좋은 재료 고르는 법, 신선도, 대체 재료
miseEnPlace         — 손질/사전 준비, 흔한 준비 실수
cookware            — { recommended, alternatives[] }
chefsNotes?         — (선택) 조리 과학/원리, <details> 펼쳐보기로 렌더링
platingAndServing   — 플레이팅과 서빙 제안
pairing             — 곁들임 추천(음료/사이드) + 왜 어울리는지
storageAndReheating — 보관 및 재가열
leftoverIdeas       — 남았을 때 활용 아이디어
closingNote         — 셰프의 마무리 한마디 (따뜻하게)
```

**배지 규칙**: `✨ 마스터클래스` 배지는 `level === 'chef'`일 때만 화면에 노출한다. masterclass 콘텐츠 자체는 모든 레벨에 들어가지만, 배지는 셰프 컬렉션에만 — 그렇지 않으면 "데일리 홈쿡 = 쉬운 요리"라는 메시지와 배지가 모순된다.

### 3.3 단계(`RecipeStep`) — 9요소
모든 단계가 9개를 다 채울 필요는 없지만, 가능하면 아래 요소들을 자연스럽게 담는다.

| 필드 | 내용 |
|---|---|
| `description` | **무엇을(WHAT) + 왜(WHY)**를 한 문장씩. "~게 핵심이에요" 같은 표현 반복 피하기 |
| `checkpoint` | 감각적 신호 — 색/향/소리/질감/타이밍 중 그 단계에 맞는 것 1~2개 |
| `warning` | 흔히 하는 실수 (선택, 위험할 때만) |
| `recoveryTip` | 그 실수를 이미 했을 때 되돌리는 법 (warning과 짝) |
| `tip` | 경험자만 아는 셰프 팁 (선택) |
| `timerSeconds` | 타이머가 의미 있는 단계에만 |
| `stepImage` | AI 생성 이미지(선택, 기존 레시피에만 주로 있음) |

## 4. 글쓰기 톤 — 반복 패턴 점검 필수

과거에 양산하다 보니 "~신호예요", "~게 핵심이에요", "이 과정에서" 같은 틀이 레시피마다 반복되어 AI가 쓴 것처럼 보인 적이 있다. 새 레시피를 쓸 때:

- 종결 표현을 의도적으로 다양하게 (~즈음이면, ~셈이니까요, ~돌아와요 등)
- 문장 길이에 리듬을 줄 것 (짧은 문장 + 긴 문장 교차)
- 인과 설명을 맥락마다 다른 구조로
- **분량은 짧고 명확하게.** 친절하게 설명하려다 길어지는 게 가장 흔한 실수다. 핵심 동작 + 이유 한 문장이면 충분하고, 부가 설명(용어 풀이, 비유 등)은 꼭 필요할 때만.
- 긴 글(셰프의 한마디 등)은 `\n\n`으로 문단을 나눠서 가독성을 챙긴다. (`RecipeBody.tsx`의 `Paragraphs` 컴포넌트가 자동으로 여러 `<p>`로 렌더링함)

## 5. 세계 요리(`world`) 작성 시

- `cuisineContext: { country, note }` 필수.
- `note`는 그 나라/지역에서 이 요리가 왜·언제·어떻게 즐겨지는지 2~3문장으로.
- 같은 식재료라도 나라를 다양하게 고른다 (예: 무는 일본/프랑스/중국, 고구마는 일본/중국/아르헨티나/서아프리카처럼 대륙을 고르게 분산).

## 6. 셰프 컬렉션(`chef`) 작성 시

- `platingGuide` 필수 — 구체적인 플레이팅 동작(어디에 무엇을 어떻게 올리는지)을 1~2문장으로.
- 같은 기법(콩소메, 카르파치오, 그라니타, 무스, 퓌레+시어드 단백질)을 여러 식재료에 반복 적용하는 건 괜찮다 — 실제 파인다이닝에서도 기법을 재료를 바꿔 반복하는 게 흔하다.

## 7. 작업 절차 (새 식재료 확장 시)

1. `python3` 스크립트로 현재 레벨별 개수 확인 (`mainIngredient` 정확매칭 기준).
2. 부족한 만큼 새 `.ts` 파일 작성 (`recipes-{식재료}-expansion.ts`), `Recipe[]` export. **파일 생성 전 동일/유사 이름 파일이 이미 존재하는지 `ls src/data/recipes-*.ts`로 확인할 것** (예: 대파=recipes-greenonion-expansion.ts, 쪽파는 이름이 겹쳐 recipes-jjokpa-expansion.ts로 분리함).
3. `src/data/recipes.ts`에 import + 배열 스프레드 추가.
4. `npx tsc --noEmit` → 타입체크.
5. `npx eslint <변경파일>` → lint.
6. 중복 id 확인: `grep -o "id: '[^']*'" src/data/recipes*.ts | sort | uniq -d`
7. `npx next build` → 빌드 확인.
8. **배포 전 항상**: 새 클론 → `months.ts`의 `blob.vercel-storage.com` 개수(287)가 유지되는지 확인 후 변경 파일만 복사 → commit & push. (과거 stale 로컬 파일을 그대로 push해서 Gemini 생성 이미지 287개를 날린 사고가 있었음 — 절대 로컬 작업 디렉토리를 그대로 덮어쓰지 말 것.)

## 8. 지금까지 완성된 식재료

**KAMIS 9개 핵심 식재료** (4단계 각 5개 이상, 무 20개 완성 포함): 감자, 사과, 무, 방울토마토, 알배추, 배, 고구마, 딸기, 오이

**추가 확장 식재료** (4단계 각 5개): 가지, 시금치, 콩나물, 애호박, 부추, 대파, 시래기, 깻잎, 미나리, 냉이, 두릅, 취나물, 표고버섯, 느타리버섯, 검은콩, 들깨, 흑미, 곶감, 바지락, 삼치, 도미, 주꾸미, 쪽파, 유채나물, 물미역, 달래, 봄동

나머지 식재료는 아직 home 1~4개뿐인 상태. 다음 확장 후보(개수 많은 순): 단호박(4), 굴(3)/단감(3)/명태(3)/전복(3)/죽순(3), 그 외 2개씩 있는 재료 다수(갈치, 갑오징어, 고구마순, 대추, 돌나물, 동치미무, 매생이, 머위, 메주콩, 멸치, 모과, 무화과, 물메기, 복숭아, 블루베리, 석류, 석이버섯, 수박, 실치, 양상추, 열무, 완두콩, 유자, 쭈꾸미, 참외, 초당옥수수, 키조개, 포도, 풋고추, 한라봉 등).

**콘텐츠 밀도 기준 (도미부터 적용, 이후 계속 유지)**: 스텝은 5~7단계로 세분화(밑간/물기제거/예열확인 등 별도 단계로 분리), 마스터클래스 각 항목은 2~3문장으로 왜 그렇게 하는지와 실패 사례까지 포함, 체크포인트는 색·냄새·소리·질감 등 감각적 신호로 구체화.
